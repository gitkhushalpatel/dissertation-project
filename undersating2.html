Project Development Roadmap

A visual guide to building the Client-Side Cryptographic Overlay System, from initial setup to final documentation.

The 6 Phases of Development

Phase 1: Core Client-Side Crypto

Establish fundamental encryption/decryption in the browser using the Web Crypto API. Focus on key derivation (PBKDF2), file encryption (AES-GCM), and key wrapping. The goal is a working, local-only prototype.

Phase 2: Backend Integration

Connect the frontend to the Python (Flask) backend. Develop API endpoints to securely store and retrieve wrapped keys and metadata in MongoDB, laying the groundwork for persistent data.

Phase 3: Cloud Storage Integration

Enable uploading and downloading of encrypted files to Google Drive. Implement OAuth 2.0 for user authorization and use the Google Drive API to manage files, linking them to keys stored in MongoDB.

Phase 4: User Experience & Recovery

Enhance the UI/UX with clear feedback, progress indicators, and user-friendly error messages. Implement a secure key recovery mechanism, such as using security questions, to improve usability.

Phase 5: Security Hardening & Testing

Fortify the system by implementing security best practices like input validation, rate limiting, and HTTPS. Conduct comprehensive unit, integration, and security testing to ensure reliability and robustness.

Phase 6: Documentation & Refinement

Prepare the project for submission. Create detailed technical documentation, a user manual, a security analysis, and performance benchmarks. Refine the UI and address any final bugs.
