The Cryptographic Core

File Encryption & Key Wrapping

1

Generate File Key

For each file, a unique, random 256-bit key (FEK) is created in your browser.

2

Encrypt File

The file's content is encrypted with its unique FEK using AES-256-GCM.

3

Wrap the Key

The FEK itself is then encrypted ("wrapped") using your Master Key (derived from your password).

4

Upload to Cloud

The encrypted file and the wrapped FEK are sent to the cloud for storage. The server sees nothing but ciphertext.

Key Derivation Function (KDF) Comparison

A KDF's job is to turn your password into a strong key. This system uses PBKDF2, but Argon2id represents the modern state-of-the-art.

Security Under the Microscope

PBKDF2 Performance vs. Security

Increasing the iteration count makes it exponentially harder for attackers to crack passwords, with a minimal delay for the legitimate user.
